Install Kde Plasma Desktop Proxmox 8.3

A guide for installing, and fixing after install critical bugs of kde plasma desktop.

Prep

Warnings

The act of installing a desktop environment onto a operation system such as proxmox is highly irregular and surely not standard practice.

However it can have it uses. an example is the factor of not requiring another computer to access the web gui.

run updates

apt update (finds the latest packages)

apt upgrade -y (installs the latest packages, -y says yes for you)

(alternative) sudo apt full-upgrade -y (similer to the prevous but will replaces and remove obsolete packages)

(optional) reboot

Configure desktop user

Method 1 (Create user & home directory)

useradd -m user-name-here (-m creates a home directory for the user, without it if you attempted to login there would be nothing to go to after login resulting in the login looping)

passwd user-name-here (passwd engages password creation)

(optional)usermod -aG sudo user-name-here (gives the user sudo privileges eg:admin)

Method 2 (Add home directory to existing user)

it is unrecommened to use the root user as the desktop user. use method 1 if you only have root user

sudo mkdir /home/user-name-here (create the Home Directory under the users name)

sudo chown username:username /home/username (change ownership of the directory to the user)

sudo usermod -d /home/username username (assigns the home directory to the user)

sudo chmod 755 /home/username (the first number is owner second number group and last number other 7 gives the owner full permissions and 5&5 gives the groups and others read & execute

sudo mkdir /home/user-name-here (Create the Home Directory)

(optional) Check the home directory with getent passwd username

Install desktop

(required) [apt install kde-plasma-desktop] Minimal Plasma desktop without many KDE applications

(recommended) [apt install kde-full] Full KDE environment] includes all apps and components

(alternative) [apt install kde-standard] (less programs then kde-full)

Fix discovery&.deb installers

[nmcli con add type dummy con-name fake ifname fake0 ip4 1.2.3.4/24 gw4 1.2.3.1] adds dummy interface so network manager deos'nt disable itself.

Fix discovery without proxmox enterpise

connect to localhost:8006 in your browser and select your node. scroll to repositories under updates disable both enterprise repositories and reload.
